Introduction

Sonic is a computer programming language I have designed. Its purpose is to facilitate the synthesis and processing of digital audio recordings, with emphasis on concise coding and rapid development. Sonic contains constructs which will be familiar to users of languages like C++ and Pascal, such as assignment statements, procedures, local variables, while loops, and if statements. However, because it is so specialized for working with digital audio, Sonic is not intended to be a general-purpose programming language.

Features of Sonic

  • Concise, powerful syntax for expressing many DSP algorithms.
  • Many useful digital audio programs can be written in less than 10 lines of code!
  • Built-in support for efficient sinewaves, sawtooths, and linear filters (IIR).
  • Language is user-extensible: write your own C++ classes and import them into Sonic programs.
  • Automatic scaling of output levels - Sonic uses temporary floating point files to hold output, then scales to fixed point before exiting.
  • It's free! I'm doing this for fun, not money.

Limitations of Sonic

  • Only the WAV audio format is currently supported. (Want your favorite format supported? Point me to a good file format specification and I'll see what I can do...)
  • No console text I/O yet. Currently you cannot write the classic "Hello World" program in Sonic, unless perhaps you want to output Morse Code!
  • Input and output audio files must be specified on the command line in a Sonic program in order to persist after program execution terminates. Programs may create temporary files to aid in creating persistent output files, but these temporary files are automatically erased when the program quits.

Installation Procedure

This is the recommended installation procedure for getting Sonic up and running on your computer. All of the files you need can be downloaded below.
  1. Create a directory on your system for Sonic, e.g. /sonic on a Unix machine, or C:\SONIC on a DOS/Windows machine.
  2. Create a directory under the Sonic directory for the runtime library, e.g., /sonic/runtime or C:\SONIC\RUNTIME.
  3. Download the file runtime.zip into the directory you created in the previous step. Unzip the contents of the file into the same directory.
  4. Create a directory for the Sonic/C++ translator source code, e.g., /sonic/src.
  5. Download the file sonicsrc.zip into the source directory from the previous step. Unzip its contents into the same directory.
  6. If you are using 32-bit Windows (95, 98, or NT), and you don't want to bother with getting the translator source code to compile, then create a directory like C:\SONIC\BIN and download the file sonicexe.zip there. Unzip the contents of this file into the same place. Add this directory to the PATH by editing your AUTOEXEC.BAT file, so that the operating system can find SONIC.EXE.
  7. Alternatively, if you are not running 32-bit Windows, you will have to compile the Sonic/C++ translator yourself. Use your favorite C++ compiler to compile and link all of the *.cpp files in /sonic/src. All of the source and header files you need come from sonicsrc.zip. You don't need the runtime.zip to compile and link the translator itself; that source code is used only to compile and link C++ source code generated by the translator. Make sure the resulting executable is in your search path so that the operating system can find it.

Download version 0.902 (beta)

  • sonicsrc.zip [50k] - Complete C++ source code to the Sonic/C++ translator. Developed as a Win32 console mode program, but should be easy to port to other operating systems like Unix, OS/2, etc. In fact, if you have to modify the source code at all to get this to compile on your system, please let me know at the email address above so that I can fix it.
  • sonicexe.zip [76k] - Contains Windows 95 / 98 / NT executable for the Sonic/C++ translator. Although this is a command-line program, it won't run under MS-DOS or Windows 3.X due to use of 32-bit instructions. If you are using a 32-bit variety of Microsoft Windows, you can download this file if you don't want to bother with compiling the C++ source in the file sonicsrc.zip above.
  • runtime.zip [14k] - You will need the source code and headers in this zip file to compile and link the output of the Sonic/C++ translator. This is in effect the runtime library of the Sonic language.

Feedback requested!

In its current state, Sonic is rather limited. I make no pretense that it is the ultimate language for its stated purpose. Nonetheless, I am excited by its potential as a digital audio programming language. I think I'm onto the germ of a good idea here. I'm publishing this language because the feedback of a community of users is invaluable for a well-rounded system. My intent is to spark interest in others interested in digital audio programming, and see if this language can become something truly valuable.

Please send any comments, questions, or suggestions about Sonic to the email address at the top of this page. I can't promise I will satisfy every request. As Sonic changes and matures as a language, there will be conflicting forces trying to pull it in different directions. I will have to make decisions on how to steer it in the "right direction", whatever that means. However, I will definitely strive to fix any bugs reported, and suggestions for future development of Sonic will influence my decisions and priorities. Ideas are always appreciated, even if I end up deciding not to use them.
