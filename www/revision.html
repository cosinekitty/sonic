Revision History of the Sonic Programming Language

by Don Cross - cosinekitty@gmail.com

This page lists changes to the Sonic programming language, along with the version number and release date of each change. Changes are listed in reverse chronological order.

Version Date Released Description of Changes
0.907 10 November 1998
  • Bug fixed in translator: non-program functions were being "forgotten" if they didn't occur in the last source module in a program with more than one source module.
  • Bug fixed in runtime library: first channel of sample read from the last buffer of a temporary input file was being set to zero. Now the correct data is returned.
  • Now the runtime library cleans up orphaned temporary files as they occur, instead of waiting until the program finishes. This keeps them from "piling up".
0.906 2 October 1998
  • Sonic/C++ translator can now operate on multiple command-line source filenames to produce a single program. This means programmers can more easily re-use code as collections of functions and global variables.
  • Minor changes to the layout of generated source code. Not so many blank lines between statements now, etc.
  • Now allow '?' to be the first dimension of an array variable passed to a function. This is analogous to '[]' in the first dimension of an array in C++.
0.905 30 September 1998
  • Added a for loop construct.
0.904 29 September 1998
  • Added array types, variables, and function parameters.
  • Increased cache size for input waves from 256 samples to 64K samples. This greatly improves runtime performance for wave assignments that have certain non-linear index expressions.
  • Fixed lexical analyzer bug: double-slash comments at the beginning of two consecutive lines caused the second comment to be not skipped over, resulting in an erroneous parser error. Yuck!
  • Eliminated unnecessary typecasts in generated C++ calls to intrinsic functions and wave expressions.
0.903 26 September 1998
  • Added Fast Fourier Transform filter pseudo-function fft.
  • Added support for passing variables to a function by reference.
0.902 21 September 1998
  • Added support for global variables.
  • Fixed a bug in which a variable of type other than import function could have more than one initializer expression enclosed in parentheses.
  • There was a bug that caused the program function to not be found if a function tried to call it, resulting in an "unknown function" error. This has been corrected.
  • There was a situation where the translator could report an "internal error" that was actually an ordinary syntax error. The wording of the error message has been changed to avoid confusing the user.
0.901 19 September 1998
  • Added the ability to linearly interpolate between samples when an input wave is subscripted with a non-integer sample index expression. This results in much smoother sound in that case. See the documentation about the new built-in constant interpolate.
  • Now an error occurs when the translator encounters more than one definition of any of the built-in constants.
  • Was not checking the data types of channel index and sample index in a wave expression. Now, if they are not of numeric type, a translator error occurs.
  • Made SonicType::operator== in the translator source be a const member function.
  • Got rid of member variables SonicParse_Program::bitsPerSample and SonicParse_Program::bitsPerSample_explicit because they were not being used anyway.
0.900 14 September 1998 First release of Sonic on the Web.
